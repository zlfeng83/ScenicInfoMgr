# AI伴游项目会话设计变更总结

本会话主要围绕 **高德地图 (AMap) 深度集成**、**地图交互增强**、**用户模拟功能** 以及 **UI 组件化配置** 进行了多项关键的设计变更与功能迭代。

## 1. 地图引擎深度集成 (AMap Integration)
- **从模拟 Canvas 转向真实地图**：移除了原有的 canvas 绘图地图逻辑，采用了 `@amap/amap-jsapi-loader` 加载高德地图 JS API 2.0。
- **3D 视野支持**：默认启用 `3D` 视图模式，设置了 `45` 度的俯仰角 (`pitch`)，增强了空间层次感。
- **自定义盖物样式**：
    - **POI Marker**：基于原生 HTML/CSS 实现了与 UI 风格高度一致的景点标记，支持名称标签显示和选中高亮。
    - **路径绘制**：实现了虚线样式的 `Polyline` 导航路径，清晰展示推荐游览线路。
- **动态大比例尺**：默认地图缩放级别调整为 `zoom: 14`（约覆盖 1 公里视野），兼顾了微观景点与宏观方位感。

## 2. 数据精准化修复 (Data Accuracy)
- **坐标校准**：
    - **故宫 (Gugong)**：校准了太和殿、乾清宫、景阳宫的经纬度，修复了 Marker 堆叠的问题。
    - **蚂蚁 A 空间 (AntSpace)**：更新为真实的杭州蚂蚁 A 空间园区坐标及其内部 POI（前台咖啡、露台花园）。
    - **西湖 (Westlake)**：注入了真实地理坐标以确保地图首次加载的准确性。

## 3. 用户位置模拟系统 (Position Simulation)
- **坐标模拟模式**：在全局状态 `useTourStore` 中引入了 `enablePosSimulation` 开关，支持真实地理维度的“导游预演”。
- **交互式位移**：
    - 用户在地图上任意位置**点击**，蓝色小人图标会移动到对应坐标。
    - 点击任一 **POI Marker**，小人会自动同步到该景点，并触发语音解说。
- **状态同步重置**：当用户在 TopBar 切换景区（如从故宫切到西湖）时，模拟坐标会自动重置为新景区的首个景点坐标，防止画面偏移。

## 4. 全局设置中心 (Settings Widget)
- **功能入口**：在 TopBar 右侧新增了设置图标（齿轮），点击弹出浮动面板。
- **动态配置项**：
    - **坐标模拟开关**：启用/禁用地图点击后的位置刷新。
    - **云端视觉识别开关**：控制是否开启后台图像识别处理。
    - **实时比例尺调整**：通过滑动条 (Slider) 动态修改 `zoom` 级别，提供从 3 到 20 级的自由视窗控制。

## 5. 数字人与 UI 体验优化
- **景区专属人设**：
    - 为故宫生成并配置了“甄嬛风格”的文化使者小谷。
    - 为蚂蚁空间配置了职场 tech 风格的小游。
    - 聊天框内的 Bot 头像与主页面的悬浮数字人实时同步。
- **图文布局增强**：修复了聊天气泡在不同屏幕尺寸下的边距填充与圆角溢出问题，提升了视觉精致感。

---
*本总结旨在记录 AI 伴游项目从基础 Demo 向具有真实地理感知能力的交互式应用转化的关键里程碑。*
